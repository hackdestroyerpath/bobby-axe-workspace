# Тех задание для Бобби.

Создать умную автоматическую торговую систему, которая:
1) Будет торговать через AIP Binance (данные введу сам).

2) ТОЛЬКО Gridbot торговля, ТОЛЬКО USDC-futers, ТОЛЬКО <=1m TF.
Для LONG / SHORT - размещение (автоматическое снятие при достижении цены SL | TP)
Для NETURAL - размещение + снятие (доп логика на снятие, т.е. выполнение запроса по хуку + логика "а что является условием снятия натуральной сетки?")

3) Параметры сеток:
- Обязательно у каждой SL + TP (определяет алгоритм).
- Обязательно у каждой ДО 7 уровней (ср. - 3-5). 7 - предел. (задается пользователем, может потом поставлю 10).
- У каждой изолированная маржа.
- У каждой сетки включать ползунок маржи на максимум (проверять каждый раз). Например 125 / 75 / 50 / 25.

Алгоритм сам определяет параметры сетки:
-Ticker
-Price_up
-Price_down
-Grids (7-2)
-SL
-TP
-LONG / SHORT / NETUR

Скорость совершения сделок (размещения сеток) - высокая.
Риск - до от банка до 10 % на одну сетку.


4) ОБЯЗАТЕЛЬНЫЙ ИСТОЧНИК ДАННЫХ ДЛЯ АНАЛИЗА (на основании анализа - заполняются параметры сетки из 3го пункта):
4.1. - Вертикальные объемы.
4.2. - Горизонтальные объемы.
4.3. - Уровни Фибоначи.
4.4. - Тики (сумма+время).
4.5. - Активность (Кол-во сделок в секунду).
4.7. - Общая активность (BTC + сектор + sp и т.д. у каждой свой вес).
4.8. - Волны Эллиота.
ДРУГИЕ ИСТОЧНИКИ АНАЛИЗА ДАННЫХ - запрещены.(запрещены для анализа - получать можно любые.).
ДЛЯ КАЖДОГО ИСТОЧНИКА ДАННЫХ - свой СУБАГЕНТ!
У КАЖДОГО ИСТОЧНИКА ДАННЫХ - свой вес. при принятии решения.


Схема такая:
1) Бот парсит данные в режиме онлайн (в полть до тиков) ВСЕХ монет Fut USDC (их там не так много) в режиме live.
2) В каждый момент времени есть вероятность (на основании анализа и сумме данных из пункта 4) открыть сетку (например) ROI > 300%  для каждой монеты, для сеток LONG / SHORT / NETUR (т.е. рассчитывается 3 вероятности, для каждого напр.)
3) Как только люлая из LONG / SHORT / NETUR вероятностей достигает поргового значения (напр > 80%) - открывается сетка. Для NETURAL сеток ставится расписание - снять сетку (при таком-то вебхуке).

То есть алгоритм в каждый момент времени рассчитывает лонг, шорт, нейтральные веротности сетки (с определенными параметрами, они как правило взаимоконвертируемы), и как только достигается высокая вероятнасть (соответвующая положительному мат. ожиданию) - открывается сетка (лонг , шорт - поставил и сверил толькок результат, в нейтральной - поставил, снял + результат).
