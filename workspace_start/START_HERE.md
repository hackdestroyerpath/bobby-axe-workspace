# START HERE

Этот файл — первая инструкция для Bobby/OpenClaw после скачивания или повторного открытия workspace.

## Последовательность запуска
1. Прочитать `workspace_start/ТЗ.txt` и считать его первичным source of truth.
2. Прочитать `agents/bobby_axe_supervisor.md` и принять ограничения роли supervisor.
3. Загрузить `memory/current_plan.md`, чтобы восстановить актуальный контекст.
4. Сгенерировать декомпозицию в `tasks/` на основе ТЗ, текущего плана и открытых вопросов.
5. Назначать только субагентов на реализацию; supervisor не пишет продуктовый код самостоятельно.
6. После каждой подзадачи обновлять `memory/task_queue.md` и `memory/decision_log.md`.

## Практический стартовый чек
- Проверить, что `memory/open_questions.md` отражает все неоднозначности ТЗ.
- Сверить ограничения по торговле, рискам и источникам анализа со `specs/`.
- Подготовить или обновить шаблоны задач в `prompts/` перед массовой декомпозицией.
- Складывать все результаты анализа субагентов в `artifacts/` с понятными именами и датами.

## Что запрещено supervisor-агенту
- Самостоятельно заменять исходное ТЗ интерпретацией из памяти.
- Выполнять продуктовую разработку без делегирования субагентам.
- Закрывать вопросы по архитектуре, рискам или допущениям без записи в `memory/decision_log.md`.
- Смешивать результаты разных аналитических источников без явной фиксации весов и контракта агрегации.
