# Bobby Axe Workspace

Этот репозиторий — стартовый workspace для OpenClaw/Bobby Axe supervisor-агента, который должен вести разработку автоматической торговой системы через декомпозицию, делегирование субагентам и фиксацию контекста в памяти.

## Назначение
- дать главному агенту предсказуемую структуру каталогов;
- сохранить исходное ТЗ как неизменяемый первичный источник требований;
- предоставить точки входа для планирования, постановки задач, накопления решений и артефактов анализа;
- минимизировать ручную реконструкцию контекста после перезапуска или передачи workspace другому агенту.

## Где лежит source of truth
- Первичный источник требований: `workspace_start/ТЗ.txt`.
- Первая инструкция после открытия workspace: `workspace_start/START_HERE.md`.
- Правила работы главного агента: `agents/bobby_axe_supervisor.md`.

## Как использовать workspace
1. Прочитать `workspace_start/ТЗ.txt` без переопределения исходных требований.
2. Прочитать `workspace_start/START_HERE.md` и следовать стартовой последовательности.
3. Открыть `agents/bobby_axe_supervisor.md`, чтобы понять роли supervisor-агента и субагентов.
4. Загрузить текущее состояние из каталога `memory/`.
5. Открыть stage-based analysis specs в `agents/analysis/`, чтобы работать по этапам расчёта, а не по списку обязательных внешних источников.
6. Сформировать атомарные задачи в каталоге `tasks/` и делегировать реализацию только субагентам.
7. Складывать промежуточные результаты в `artifacts/`, а нормативные описания — в `specs/` и `checklists/`.

## Актуальные analysis/spec документы
- `agents/analysis/feature_extraction_agent.md` — контракт извлечения микро-признаков из тиков и market snapshots.
- `agents/analysis/probability_engine_agent.md` — контракт вычисления вероятностей `LONG` / `SHORT` / `NEUTRAL`.
- `agents/analysis/grid_synthesis_agent.md` — контракт синтеза параметров сетки и execution proposal.
- `agents/analysis/neutral_lifecycle_agent.md` — контракт сопровождения и снятия `NEUTRAL`.
- Исторические source-based документы архивированы в `agents/archive/data_sources_legacy/` и не считаются актуальным нормативным слоем.

## Ограничения, зафиксированные в workspace
- `workspace_start/ТЗ.txt` считается неизменяемым первичным источником требований; любые интерпретации должны ссылаться на него.
- Главный агент не пишет продуктовый код сам: он планирует, назначает исполнителей, контролирует и проверяет результат.
- Аналитический слой должен быть описан через этапы расчёта и контракты модулей, а не через обязательный allowlist внешних источников.
- Обновление состояния после каждой подзадачи обязательно: минимум `memory/task_queue.md` и `memory/decision_log.md`.
- Структура сделана минимальной; детали должны дорастать через `specs/`, `tasks/`, `prompts/` и `artifacts/`.
