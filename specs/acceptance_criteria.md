# Acceptance Criteria

## Для этапа проектирования
- source of truth определен и не дублируется в противоречивой форме;
- роли supervisor и субагентов формализованы;
- память workspace содержит план, очередь задач, лог решений и открытые вопросы;
- есть стартовые документы для архитектуры, ограничений и контрактов.

## Для каждой атомарной задачи
- у задачи есть явный owner;
- зафиксирован ожидаемый результат;
- определен способ проверки;
- статус отражен в `memory/task_queue.md`.
