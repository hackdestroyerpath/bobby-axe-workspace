# System Architecture

## Цель документа
Описать целевую архитектуру торговой системы на уровне модулей, потоков данных и зон ответственности субагентов.

## Минимальный каркас
- ingestion live-данных по всем USDC futures инструментам;
- слой аналитических субагентов по каждому разрешенному источнику;
- агрегатор вероятностей long / short / neutral;
- модуль генерации параметров grid-сетки;
- исполнительный контур постановки/снятия сеток;
- контур контроля риска и журналирования.

## Следующий шаг
После первой архитектурной сессии supervisor должен дополнить документ диаграммами, интерфейсами и критическими зависимостями.
