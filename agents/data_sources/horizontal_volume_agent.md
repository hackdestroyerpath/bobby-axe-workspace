# Subagent: Horizontal Volume Agent

## Допустимые входные данные
- volume profile по ценовым уровням за заданное lookback-окно;
- `POC`, `VAH`, `VAL`, high/low диапазона;
- цена последней сделки и текущий торговый диапазон;
- временная метка расчёта.

## Что считает субагент
Субагент определяет области концентрации ликвидности и принятия цены, а также вероятность реакции от объёмных узлов и пустот.

## Возвращаемые признаки и метрики
- `point_of_control_distance`
- `value_area_position`
- `low_volume_gap_distance`
- `acceptance_rejection_state`
- `volume_node_asymmetry`
- `direction_candidate`
- `recommended_grid_bounds`
- `suggested_tp`
- `suggested_sl`
- `invalidators`

## Формат confidence score
```json
{
  "score": 0.0,
  "scale": "0.0-1.0",
  "label": "low|medium|high"
}
```
`score` отражает качество структурного уровня и степень подтверждения реакции цены на профиль объёма.

## Запрещённые данные для принятия решения
- тиковый поток как самостоятельный источник;
- показатели funding/open interest, если они не входят в профиль объёма;
- внешние корреляции по другим рынкам;
- любые неразрешённые аналитические источники.

## Допустимая частота обновления
- каждые `30s`;
- обязательно после существенного изменения диапазона профиля или перестройки `POC`.
