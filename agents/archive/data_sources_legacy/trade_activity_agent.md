# Subagent: Trade Activity Agent

## Допустимые входные данные
- количество сделок за окно времени;
- суммарный размер сделок за окно времени;
- средний размер сделки;
- текущая цена и timestamp.

## Что считает субагент
Субагент измеряет активность торгового потока: ускорение/затухание числа сделок, изменение среднего размера сделки и подтверждение режима участия рынка.

## Возвращаемые признаки и метрики
- `trades_per_sec`
- `avg_trade_size`
- `activity_zscore`
- `large_trade_share`
- `activity_regime`
- `direction_candidate`
- `recommended_grid_bounds`
- `suggested_tp`
- `suggested_sl`
- `invalidators`

## Формат confidence score
```json
{
  "score": 0.0,
  "scale": "0.0-1.0",
  "label": "low|medium|high"
}
```
`score` отражает выраженность режима активности и то, насколько он подтверждает направленное или нейтральное состояние.

## Запрещённые данные для принятия решения
- order book depth и скрытая ликвидность;
- внешние новости;
- волновая и фибоначчи-разметка;
- любые источники, не утверждённые для decision layer.

## Интерпретация внутри live loop
- расчёт должен обновляться внутри секундного decision cycle;
- всплески `trades_per_sec` могут вызывать внутренний пересмотр, но не заменяют обязательный секундный refresh;
- нельзя трактовать этот документ как разрешение на независимый update cadence медленнее 1 секунды.
