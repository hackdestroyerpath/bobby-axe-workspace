# Subagent: Elliott Wave Agent

## Допустимые входные данные
- подтверждённые swing-структуры цены;
- последовательность экстремумов и коррекций на таймфреймах (`15m`, `1h`, `4h`);
- допустимая внутренняя разметка волн по формализованным правилам;
- timestamp расчёта и идентификатор инструмента.

## Что считает субагент
Субагент формирует рабочую волновую гипотезу, определяет текущую фазу импульса/коррекции и оценивает сценарное направление с учётом правил валидности волновой структуры.

## Возвращаемые признаки и метрики
- `wave_count_label`
- `active_wave_phase`
- `impulse_vs_correction_bias`
- `wave_invalidation_level`
- `pattern_clarity_score`
- `direction_candidate`
- `recommended_grid_bounds`
- `suggested_tp`
- `suggested_sl`
- `invalidators`

## Формат confidence score
```json
{
  "score": 0.0,
  "scale": "0.0-1.0",
  "label": "low|medium|high"
}
```
`score` зависит от чистоты волновой структуры, числа альтернативных сценариев и близости к уровню инвалидации.

## Запрещённые данные для принятия решения
- субъективная ручная разметка без формализованных правил;
- новости и макро-триггеры;
- тиковый шум как замена волновой структуры;
- сигналы нерелевантных аналитических модулей.

## Допустимая частота обновления
- каждые `300s`;
- внеочередное обновление при подтверждении новой волны или пробое уровня инвалидации.
