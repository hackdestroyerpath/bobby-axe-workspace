# Subagent: Vertical Volume Agent

## Допустимые входные данные
- OHLCV-бары по инструменту на разрешённых таймфреймах (`1m`, `5m`, `15m`, `1h`);
- агрегированный объём по барам;
- rolling baseline объёма за историческое окно;
- метки времени и идентификатор инструмента.

## Что считает субагент
Субагент оценивает вертикальный объём во времени: ускорение объёма, отклонение от baseline и подтверждение направленного импульса ценовым баром.

## Возвращаемые признаки и метрики
- `relative_volume_ratio`
- `volume_zscore`
- `expansion_streak_bars`
- `price_volume_alignment`
- `direction_candidate`
- `recommended_grid_bounds`
- `suggested_tp`
- `suggested_sl`
- `invalidators`

## Формат confidence score
```json
{
  "score": 0.0,
  "scale": "0.0-1.0",
  "label": "low|medium|high"
}
```
`score` отражает силу аномалии объёма и согласованность с движением цены.

## Запрещённые данные для принятия решения
- новостные ленты и макро-события;
- order book вне агрегированного барового объёма;
- ручные комментарии аналитика;
- сигналы неразрешённых индикаторов.

## Допустимая частота обновления
- базовая публикация: каждые `15s`;
- форсированное обновление: при закрытии нового бара рабочего таймфрейма.
