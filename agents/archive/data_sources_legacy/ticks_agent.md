# Subagent: Ticks Agent

## Допустимые входные данные
- поток последних сделок/ticks;
- timestamp каждого тика;
- цена тика, размер сделки, направление агрессора при наличии;
- агрегированное окно наблюдения до `60s` как rolling window внутри секундного цикла.

## Что считает субагент
Субагент анализирует микроструктуру: скорость прихода тиков, направленный перекос, серию агрессивных принтов и краткосрочное истощение импульса.

## Возвращаемые признаки и метрики
- `tick_rate_per_sec`
- `signed_tick_imbalance`
- `micro_burst_score`
- `mean_intertick_ms`
- `micro_exhaustion_flag`
- `direction_candidate`
- `recommended_grid_bounds`
- `suggested_tp`
- `suggested_sl`
- `invalidators`

## Формат confidence score
```json
{
  "score": 0.0,
  "scale": "0.0-1.0",
  "label": "low|medium|high"
}
```
`score` отражает силу и устойчивость микроструктурного импульса в пределах короткого окна.

## Запрещённые данные для принятия решения
- свечные паттерны старших таймфреймов как основной драйвер;
- ручная разметка без тикового подтверждения;
- внешние рыночные индексы;
- сигналы субагентов вне allowlist.

## Интерпретация внутри live loop
- расчёт должен обновляться не реже одного раза в секунду при наличии тикового потока;
- event-driven micro-burst может инициировать внутренний пересмотр, но не заменяет обязательный секундный refresh;
- любые окна наблюдения трактуются как rolling windows внутри секундного decision cycle.
