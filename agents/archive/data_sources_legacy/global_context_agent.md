# Subagent: Global Context Agent

## Допустимые входные данные
- разрешённые бенчмарки широкого рынка и кросс-активного контекста;
- относительная динамика BTC и доминирующего рыночного режима;
- агрегированные индексы риска/аппетита, явно включённые в policy системы;
- timestamp каждого входного снапшота.

## Что считает субагент
Субагент определяет глобальный рыночный режим и контекст, в котором оценивается локальный инструмент: risk-on, risk-off, смешанный или нейтральный фон.

## Возвращаемые признаки и метрики
- `benchmark_return_dispersion`
- `context_regime`
- `btc_lead_lag_bias`
- `cross_market_alignment`
- `systemic_risk_flag`
- `direction_candidate`
- `recommended_grid_bounds`
- `suggested_tp`
- `suggested_sl`
- `invalidators`

## Формат confidence score
```json
{
  "score": 0.0,
  "scale": "0.0-1.0",
  "label": "low|medium|high"
}
```
`score` отражает степень согласованности глобального контекста и применимость этого фона к текущему инструменту.

## Запрещённые данные для принятия решения
- нерелевантные социальные сигналы, слухи и текстовые мнения;
- ручная фундаментальная интерпретация без формализованного источника;
- данные из неописанных аналитических каналов;
- прямое переопределение локального сигнала без контрактного объяснения.

## Допустимая частота обновления
- каждые `60s`;
- внеочередное обновление при смене глобального режима или резком росте системного риска.
