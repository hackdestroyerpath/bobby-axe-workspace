# Analysis Module: Feature Extraction Agent

## Назначение
Модуль извлекает из live tick stream и краткоживущих рыночных буферов микро-признаки, которые нужны для последующего вероятностного расчёта. Документ описывает **этап вычисления**, а не перечень обязательных внешних аналитических школ или источников.

## Роль в pipeline
`ingestion -> feature extraction -> probability engine -> grid synthesis -> neutral lifecycle`

Этот модуль не публикует готовое торговое направление и не должен навязывать downstream-слоям фиксированную интерпретацию рынка. Его задача — подготовить воспроизводимый набор признаков и оценок качества данных.

## Входы
- live ticks / last trades с timestamp;
- краткие rolling windows по цене, объёму, интенсивности сделок и микроструктуре;
- нормализованные market snapshots из ingestion;
- служебные метаданные качества потока: gaps, lag, desync, stale-state flags.

## Что делает модуль
- нормализует входной tick stream для расчётов в коротких окнах;
- извлекает микро-признаки о давлении, дисбалансе, ускорении, затухании, расширении/сжатии диапазона и иных допустимых паттернах поведения цены и сделок;
- вычисляет derived state, пригодный для дальнейшего вероятностного вывода;
- оценивает свежесть и пригодность признаков для текущего decision cycle;
- фиксирует деградации, пропуски и неоднозначности входных данных.

## Контракт результата
Модуль должен публиковать артефакт `feature snapshot`, содержащий:
- `feature_snapshot_id`;
- `symbol`;
- `timestamp`;
- `feature_window` или описание расчётного окна;
- `derived_features[]` — список числовых и категориальных признаков;
- `data_quality` — свежесть, пропуски, статус деградации;
- `trace_refs` — ссылки на входные snapshots и calculation trace.

Каждый элемент `derived_features[]` должен включать:
- `name` — стабильное имя признака;
- `value` — числовое или категориальное значение;
- `unit` — единица измерения или `none`;
- `freshness_sec` — возраст расчёта;
- `interpretation` — краткое объяснение, что описывает признак, без жёсткой торговой директивы.

## Ограничения
- Нельзя требовать фиксированный набор индикаторов, конкретную школу анализа или заранее утверждённый список внешних аналитических источников.
- Нельзя подменять probability engine публикацией `LONG` / `SHORT` / `NEUTRAL` как окончательного решения.
- Нельзя скрывать деградацию данных: missing/stale features обязаны быть явно отражены в `data_quality`.
- Любой признак должен быть вычислим из допустимых рыночных и системных данных и воспроизводим в audit trail.

## Fail-safe поведение
- При потере свежести tick stream модуль помечает feature snapshot как `degraded` или `stale`.
- При частичной недоступности входов модуль публикует только те признаки, которые можно рассчитать надёжно, и указывает missing features.
- Если расчётное окно неконсистентно, модуль возвращает безопасный артефакт с причинами деградации вместо молчаливого пропуска.
