# Analysis Module: Probability Engine Agent

## Назначение
Модуль преобразует `feature snapshot` и market state в нормализованное распределение вероятностей `LONG`, `SHORT`, `NEUTRAL`. Документ описывает этап вероятностного вывода, а не фиксирует конкретную математическую модель, список сигналов или веса заранее заданных аналитических школ.

## Роль в pipeline
`ingestion -> feature extraction -> probability engine -> grid synthesis -> neutral lifecycle`

## Входы
- `feature snapshot` из `agents/analysis/feature_extraction_agent.md`;
- market state snapshots и служебный контекст исполнения;
- policy-ограничения по риску, качеству данных и допустимому уровню неопределённости;
- исторические/внутренние буферы, если они нужны для нормализации и устойчивости вывода.

## Что делает модуль
- проверяет полноту, свежесть и согласованность входных признаков;
- агрегирует признаки в вероятностную модель выбора состояния рынка;
- рассчитывает `P(LONG)`, `P(SHORT)`, `P(NEUTRAL)`;
- нормализует распределение и вычисляет агрегированную уверенность;
- формирует reason codes, объясняющие доминирующий исход и факторы деградации;
- переводит решение в безопасный `NEUTRAL`, когда направленный вывод недостаточно надёжен.

## Контракт результата
Модуль должен публиковать `probability snapshot`, содержащий:
- `decision_id`;
- `symbol`;
- `timestamp`;
- `input_feature_snapshot_id`;
- `probabilities.long`;
- `probabilities.short`;
- `probabilities.neutral`;
- `confidence.score` и `confidence.label`;
- `dominant_outcome`;
- `reason_codes[]`;
- `degradation_flags[]`;
- `trace_refs`.

## Обязательные инварианты
- Все три вероятности находятся в диапазоне `0.0..1.0`.
- Сумма вероятностей равна `1.0` с допустимой технической погрешностью.
- `dominant_outcome` выбирается только после проверки качества данных и confidence policy.
- При недостаточной уверенности модуль обязан повышать вероятность `NEUTRAL` вместо искусственного форсирования directional-исхода.

## Ограничения
- Нельзя жёстко закреплять обязательные индикаторы, школы анализа или перечни внешних аналитических модулей как единственный допустимый способ расчёта.
- Нельзя публиковать probability snapshot без объяснимости: требуются `reason_codes` и traceability до feature layer.
- Нельзя использовать частично повреждённый feature snapshot без явной деградации confidence.

## Fail-safe поведение
- Если входные признаки stale или неполны, модуль снижает confidence и повышает вероятность `NEUTRAL`.
- Если нормализация или проверки целостности не проходят, модуль публикует безопасный отказ для downstream-слоёв.
- Если несколько внутренних логик дают противоречивые выводы, модуль обязан отразить конфликт через confidence/degradation, а не скрывать его.
